Testing Structure


This modules is a very situational module that requires both unreadable mac addresses, Juniper devices and other factors to ensure high code coverage. The test was performed by monitoring coverage as we used the application. Below I will break down what values and actions we took to get the results described above.
Using a local area network we scanned it with both a live Juniper device on it as well as several other devices including a thrid-party USB Wi-Fi adapter. When the scan was completed we interacted with each node shown to give us the Mac and IP address of each. The USB Wi-Fi adapter wasn't registered correctly under the IEEE so no vendor was shown for it. Once we identified each device and compared it against what was actually live we cleared the arp cache and tried to get the Mac address of any address again. This time we got "N/A" for each as expected.
With the network scanned we tried to log into 1 offline device and 2 live devices , A phone and the Juniper firewall. When we tried to log into the offline device the application first checked if it was a juniper device. As it was offline it couldn't find it so by default it told us the device was unsupported. When we checked the 2 live devices the application didn't allow us to log into the phone as it wasn't a Juniper device (through looking up the mac) and we were able to successfully connect to the firewall and pull down the relevant data. Once that was complete we refreshed the page and closed it down.
